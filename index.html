How It Works

Model Pipeline

From raw patient review to 3-class sentiment prediction — five clear stages.

User Input
Patient pastes or types a hospital review into the interface.
Preprocessing
Noise removal, lowercasing, negation handling, stemming.
Tokenization
WordPiece tokenizer splits text into subword tokens for BERT.
BERT Encoding
bert-base-uncased encodes contextual embeddings via 12 transformer layers.
3-Class Output
Softmax head outputs Positive / Neutral / Negative with confidence.
About the Model

Model Architecture

A fine-tuned BERT model trained for 3-class hospital review sentiment classification.

Architecture

bert-base-uncased — 12 transformer encoder layers, 768 hidden dimensions, 12 attention heads, 110M parameters. Fine-tuned with a 3-class softmax classification head.

Dataset

10,000 curated hospital reviews derived from Yelp healthcare businesses, labelled across Positive, Neutral, and Negative categories with rigorous cleaning and balancing.

Training Techniques

  • WordPiece Tokenization
  • Padding & Truncation (512 tokens)
  • Layer Freezing (first 8 layers)
  • L1 Regularization
  • Transfer Learning
  • AdamW Optimizer

Model Reliability

Validated on a held-out test split with stratified sampling across all 3 classes. Tested for class imbalance robustness and edge cases including medical jargon and mixed-sentiment reviews.
